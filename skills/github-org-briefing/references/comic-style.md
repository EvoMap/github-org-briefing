# Comic Style Reference

Use this fixed style for GitHub organization briefing images:

- Format: one tall 4-6 panel comic explainer infographic.
- Tone: lively, witty, optimistic, and engineer-friendly.
- Visual metaphors: repositories as busy workbenches, commits as message bubbles, releases as launch moments, warnings as a watch desk.
- Accuracy rule: the comic may add playful visual framing, but all text and metrics must come from the generated report.
- Text rule: generated image backgrounds should not contain readable words; deterministic overlay text from the CLI carries the facts.
- Palette: bright editorial colors with strong ink outlines and high contrast text panels.
- Layout: title at top, panel grid in the middle, source note at bottom.
